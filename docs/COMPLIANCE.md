# Compliance Model

## SSS-2

- Public compliance controls on-chain (blacklist + seizure)
- Transfer enforcement via transfer-hook checks

## SSS-3

- Privacy-first transfers in shielded domain
- "Compliance at the Boundary": checks at shield/unshield edges
- Viewing-key hierarchy for selective disclosure:
  - `issuer`
  - `compliance`
  - `auditor`

This model supports private-by-default transactions while preserving audit and sanctions workflows where required.
