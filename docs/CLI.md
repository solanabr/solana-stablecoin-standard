# sss-token CLI Reference

`sss-token` is the command-line interface for managing Solana Stablecoin Standard tokens. It wraps the `@stbr/sss-sdk` with shell-friendly commands covering the full token lifecycle: initialization, minting, burning, compliance, and administration.

---

## Installation

```bash
# From the repository (development)
cd cli && npm install && npm run build
npm link   # makes sss-token available globally

# Or run directly without installing
node cli/dist/index.js <command>
```

---

## Global Options

These options apply to every command and can also be set via environment variables:

| Option              | Environment Variable    | Default                              | Description                      |
|---------------------|-------------------------|--------------------------------------|----------------------------------|
| `--rpc <url>`       | `ANCHOR_PROVIDER_URL`   | `https://api.devnet.solana.com`      | Solana RPC endpoint              |
| `--keypair <path>`  | `ANCHOR_WALLET`         | `~/.config/solana/id.json`           | Path to authority keypair JSON   |

The keypair file must be a JSON array of 64 bytes (standard Solana format, as generated by `solana-keygen new`). The active keypair is used as the transaction signer — the role it holds determines which operations it can perform.

```bash
# Use a custom RPC and keypair for all commands
sss-token --rpc https://api.mainnet-beta.solana.com \
          --keypair /secrets/authority.json \
          <command>

# Or set globally for the session
export ANCHOR_PROVIDER_URL=https://api.mainnet-beta.solana.com
export ANCHOR_WALLET=/secrets/authority.json
sss-token <command>
```

## Commands

### `init` — Initialize a new stablecoin

Creates a Token-2022 mint with the selected preset and initializes all SSS PDAs.

```
sss-token init [options]
```

| Option               | Required | Description                                                   |
|----------------------|----------|---------------------------------------------------------------|
| `--preset <preset>`  | Yes      | `sss-1` or `sss-2`                                           |
| `--name <name>`      | Yes      | Token name, max 32 characters                                 |
| `--symbol <symbol>`  | Yes      | Token symbol, max 10 characters                               |
| `--uri <uri>`        | Yes      | Metadata URI (HTTPS link to JSON), max 200 characters         |
| `--decimals <n>`     | No       | Decimal places 0–9 (default: 6)                               |
| `--max-supply <n>`   | No       | Max mintable supply in base units (default: 0 = unlimited)    |
| `--minter <address>` | No       | Minter wallet address (default: signing keypair)              |
| `--minter-quota <n>` | No       | Per-epoch minter quota in base units (default: 0 = unlimited) |

**Example — SSS-1:**
```bash
sss-token init \
  --preset sss-1 \
  --name "ACME USD" \
  --symbol "AUSD" \
  --uri "https://acme.finance/ausd-metadata.json" \
  --decimals 6 \
  --max-supply 1000000000000000
```

**Example — SSS-2 with separate minter keypair:**
```bash
sss-token init \
  --preset sss-2 \
  --name "Regulated USD" \
  --symbol "RUSD" \
  --uri "https://bank.example.com/rusd-metadata.json" \
  --decimals 6 \
  --max-supply 500000000000000 \
  --minter 7xKXtG7MNYkZ9jMRVsRqpTNjJjEbGCnLGbEkqNfVWdGx \
  --minter-quota 10000000000000
```

**Output:**
```

  Name: Regulated USD
  Symbol: RUSD
  Authority: 9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM
```

### `mint` — Mint tokens

Mints tokens to a recipient's associated token account. The recipient's ATA must already exist; create it first with `spl-token create-account <mint> --owner <wallet>` or use `token.getOrCreateAta()` in the SDK.

```
sss-token mint --mint <address> --to <address> --amount <n>
```

| Option            | Required | Description                          |
|-------------------|----------|--------------------------------------|
| `--mint <address>` | Yes     | Mint address                         |
| `--to <address>`   | Yes     | Destination token account address    |
| `--amount <n>`     | Yes     | Amount in base units                 |

```bash
# Mint 1,000 RUSD (6 decimals)
sss-token mint \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --to DpRueBHHhrquxs4BTd8beMy7W2cAEpsu8crznUB4fFCi \
  --amount 1000000000
```

---

### `burn` — Burn tokens

Burns tokens from a source token account. The calling keypair must be the `burner` or `master_authority`.

```
sss-token burn --mint <address> --from <address> --amount <n>
```

| Option              | Required | Description                         |
|---------------------|----------|-------------------------------------|
| `--mint <address>`  | Yes      | Mint address                        |
| `--from <address>`  | Yes      | Source token account to burn from   |
| `--amount <n>`      | Yes      | Amount in base units                |

```bash
sss-token burn \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --from DpRueBHHhrquxs4BTd8beMy7W2cAEpsu8crznUB4fFCi \
  --amount 500000000
```

---

### `freeze` — Freeze a token account

Prevents the account holder from sending or receiving tokens. The calling keypair must be `master_authority` or `pauser`.

```
sss-token freeze --mint <address> --account <address>
```

```bash
sss-token freeze \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --account DpRueBHHhrquxs4BTd8beMy7W2cAEpsu8crznUB4fFCi
```

---

### `thaw` — Unfreeze a token account

Restores normal operation for a frozen account. The calling keypair must be `master_authority` or `pauser`.

```
sss-token thaw --mint <address> --account <address>
```

```bash
sss-token thaw \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --account DpRueBHHhrquxs4BTd8beMy7W2cAEpsu8crznUB4fFCi
```

---

### `pause` — Pause all minting and burning

Sets the global pause flag. Subsequent `mint` and `burn` calls via SSS instructions will fail. The calling keypair must be `master_authority` or `pauser`.

```
sss-token pause --mint <address>
```

```bash
sss-token pause --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm
```

---

### `unpause` — Resume operations

Clears the global pause flag. The calling keypair must be `master_authority` or `pauser`.

```
sss-token unpause --mint <address>
```

```bash
sss-token unpause --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm
```

### `blacklist add` — Add address to blacklist (SSS-2 only)

Creates an on-chain `BlacklistEntry` PDA. The transfer hook will block any transfer involving this wallet. The calling keypair must be `blacklister` or `master_authority`.

```
sss-token blacklist add --mint <address> --address <wallet> [--reason <n>]
```

| Option              | Required | Description                                       |
|---------------------|----------|---------------------------------------------------|
| `--mint <address>`  | Yes      | Mint address                                      |
| `--address <wallet>`| Yes      | Wallet address to blacklist (not token account)   |
| `--reason <n>`      | No       | Reason code 0–255 (default: 0 = unspecified)      |

```bash
sss-token blacklist add \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --address BadActorWa11et111111111111111111111111111111 \
  --reason 1
```

---

### `blacklist remove` — Remove address from blacklist (SSS-2 only)

Closes the `BlacklistEntry` PDA and returns the rent to the calling wallet. The calling keypair must be `blacklister` or `master_authority`.

```
sss-token blacklist remove --mint <address> --address <wallet>
```

```bash
sss-token blacklist remove \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --address BadActorWa11et111111111111111111111111111111
```

---

### `blacklist check` — Check if an address is blacklisted

Queries the chain for a `BlacklistEntry` PDA and reports the result. No signing required.

```
sss-token blacklist check --mint <address> --address <wallet>
```

```bash
sss-token blacklist check \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --address SomeWa11et111111111111111111111111111111111
# NOT blacklisted
```

---

### `seize` — Seize tokens using permanent delegate (SSS-2 only)

Transfers tokens from a holder's account to a destination account without the holder's consent. The calling keypair must be `seizer` or `master_authority`.

```
sss-token seize --mint <address> --from <token-account> --to <token-account> --amount <n>
```

| Option             | Required | Description                              |
|--------------------|----------|------------------------------------------|
| `--mint <address>` | Yes      | Mint address                             |
| `--from <address>` | Yes      | Source token account (holder's ATA)      |
| `--to <address>`   | Yes      | Destination token account (recovery ATA) |
| `--amount <n>`     | Yes      | Amount in base units                     |

```bash
sss-token seize \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --from BadActorATA11111111111111111111111111111111111 \
  --to RecoveryATA1111111111111111111111111111111111111 \
  --amount 5000000000
```

### `minters list` — Show minter configuration

Displays the current minter address, quota, and epoch usage. No signing required.

```
sss-token minters list --mint <address>
```

```bash
sss-token minters list --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm
# Minter: 7xKXtG7MNYkZ9jMRVsRqpTNjJjEbGCnLGbEkqNfVWdGx
# Quota: 10000000000000
# Minted this epoch: 1000000000000
```

---

### `minters add` — Set a new minter

Replaces the current minter with a new address and optionally updates the quota. Setting a new quota also resets the epoch usage counter. The calling keypair must be `master_authority`.

```
sss-token minters add --mint <address> --minter <address> [--quota <n>]
```

```bash
sss-token minters add \
  --mint Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm \
  --minter NewMinterKeypair11111111111111111111111111111 \
  --quota 20000000000000
```

## Example Workflows

### Full SSS-1 deployment on mainnet

```bash
export ANCHOR_PROVIDER_URL=https://api.mainnet-beta.solana.com
export ANCHOR_WALLET=/secure/master-authority.json

# 1. Initialize
sss-token init \
  --preset sss-1 \
  --name "ACME USD" \
  --symbol "AUSD" \
  --uri "https://acme.finance/ausd.json" \
  --max-supply 1000000000000000

# Note the mint address from output, e.g.: MINT=Abc123...

# 2. Create recipient ATA (standard spl-token tooling)
spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPGA1WymbbVQnDBtzdeyz \
  create-account $MINT --owner <recipient-wallet>

# 3. Mint initial supply
sss-token mint \
  --mint $MINT \
  --to <recipient-ata> \
  --amount 100000000000000

# 4. Verify
sss-token status --mint $MINT
```

### SSS-2 sanctions enforcement workflow

```bash
export ANCHOR_WALLET=/secure/compliance-officer.json
MINT=Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm

# 1. Block the sanctioned wallet immediately
sss-token blacklist add \
  --mint $MINT \
  --address <sanctioned-wallet> \
  --reason 1

# 2. Confirm it is blocked
sss-token blacklist check --mint $MINT --address <sanctioned-wallet>

# 3. Switch to seizer keypair for asset recovery
export ANCHOR_WALLET=/secure/seizer.json

sss-token seize \
  --mint $MINT \
  --from <sanctioned-ata> \
  --to <recovery-ata> \
  --amount <amount>
```

### Emergency pause and resume

```bash
export ANCHOR_WALLET=/secure/pauser.json
MINT=Aaw7RSm8fXXAfvDy9wS1FimFEmnHkzYtDmZCRgZg9pVm

# Pause immediately
sss-token pause --mint $MINT

# Investigate and resolve the issue...

# Resume
sss-token unpause --mint $MINT

# Verify status
sss-token status --mint $MINT
```

## Exit Codes

| Code | Meaning                                             |
|------|-----------------------------------------------------|
| 0    | Success                                             |
| 1    | Error (invalid input, RPC failure, program error)   |

All errors are printed to stderr with a descriptive message before exiting with code 1.
